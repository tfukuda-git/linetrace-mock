よくある課題

LINE 経由のコンバージョン、
広告に返せていますか？

計測ツールを入れていない状態では、友だち追加もその後の LINE 内アクションも、LINE 経由の Web 購入も、どれも広告に返っていません。これが多くのアカウントで起きている計測ロスです。

ツール未導入だと、ここが全部ロスになる

STEP 1

友だち追加

広告 → LP → LINE 友だち追加
✕ 広告に返らない
STEP 2

LINE 内アクション

フォーム回答・リッチメニュータップ
✕ 広告に返らない
STEP 3

Web で購入・予約

LINE 経由で Web → 購入・予約・問い合わせ
✕ 広告に返らない
なぜ計測ロスが起きるのか
広告 → LP までは追えても、LINE を挟んだ瞬間にセッションが切れるからです。LINE アプリ内ブラウザは LP で保存した Cookie（広告クリック ID）を引き継がないため、友だち追加やその後のアクションが「どの広告から来たか」と繋がらず、広告側に何も返せません。
結果、広告の AI（自動入札）が「本当に買ってくれるユーザー」を学習できず、配信最適化が止まり、CPA が高騰します。

※ 媒体CV「500」は広告媒体がクリックベースで数えた件数。実際の友だち追加（120）とズレるのはこのため。数値は典型例。

解決策

3 つのコンバージョンを、
まとめて広告に返します

友だち追加 / LINE 内 CV / LINE 経由 Web CV — どれも独立したイベントとして Meta・Google・Yahoo に Conversions API で送信。3 つすべてに対応します。

3 つのイベント、すべて対応

友だち追加 CV

LIFF や認証画面を経由せず、Cookie で広告クリック ID（gclid / fbclid / yclid）を保持。実追加数とのズレを抑えて CAPI 送信。

LINE 内 CV

LINE 内でのフォーム回答完了・リッチメニュー導線などのアクションを CV としてキャッチ。キーワードやクイックリプライで設定。

LINE 経由 Web CV

LINE から Web に戻った後の購入・予約・問い合わせを、元の広告クリックに紐付けて返却。LINE を跨いだアトリビューションが成立。

FAQ

よくあるご質問

使わない月の広告アカウント連携費はどうなりますか？

ダッシュボードから広告アカウント連携ごとに一時停止できます。停止中はその連携の CAPI 送信が止まり、その月の連携費（¥3,000）は課金対象外になります（計測・ダッシュボード閲覧は継続）。基本料金 ¥10,000 は契約中は継続します。繁忙期だけ連携し、オフシーズンは停止する、といった季節運用に対応しています。

既存の Meta Pixel・GA4 はそのままで使えますか？

はい。独立したカスタムイベントとして送信するため、既存の計測数値を汚さず、広告の機械学習用データだけを安全に追加できます。

既存の LINE 拡張ツール（エルメ・Lステップ等）と併用できますか？

はい、併用可能です。Linetrace は計測専用ツールで、Webhook を中継して既存ツールに転送する設計です。

導入にどのくらい時間がかかりますか？
