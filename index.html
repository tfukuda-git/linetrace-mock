よくある質問

パスワードを教える必要がありますか？

いいえ。各プラットフォームの正規の権限共有機能（招待・パートナー追加）を使います。パスワードやログイン情報をお預かりすることはありません。

設定後に権限を削除しても大丈夫ですか？

はい。LINE OAMとGTMの権限は、設定完了後に削除いただいて構いません。Meta BMのパートナー共有のみ、CAPI送信のために維持が必要です。

どのくらいで完了しますか？

権限共有の確認後、通常1〜2営業日で全設定を完了します。完了後にメールでお知らせし、ダッシュボードが自動で有効になります。

有料プラン移行後も代行してもらえますか？

トライアル期間中の初回設定は無料です。有料プラン移行後は、広告アカウントの追加やCAPI再設定などを月額プラン内でサポートします。

よくある課題

LINE 経由のコンバージョン、
広告に返せていますか？

計測ツールを入れていない状態では、友だち追加もその後の LINE 内アクションも、LINE 経由の Web 購入も、どれも広告に返っていません。

ツール未導入だと、ここが全部ロスになる

STEP 1

友だち追加

広告→LP→LINE友だち追加
✕ 広告に返らない
STEP 2

LINE 内アクション

フォーム回答・リッチメニュータップ
✕ 広告に返らない
STEP 3

Web で購入・予約

LINE経由でWeb→購入・予約
✕ 広告に返らない
なぜ計測ロスが起きるのか
広告→LPまでは追えても、LINEを挟んだ瞬間にセッションが切れるからです。LINEアプリ内ブラウザはLPで保存したCookie（広告クリックID）を引き継がないため、広告側に何も返せません。
結果、広告のAI（自動入札）が「本当に買ってくれるユーザー」を学習できず、配信最適化が止まり、CPAが高騰します。

Before：説明できない

媒体CV 500

LINE実追加 120

→ 乖離で最適化も説明も不能

→

After：実測できる

媒体CV 500

LINE実追加 480

→ CAPI連携で誤差解消・説明可能

解決策

3 つのコンバージョンを、
まとめて広告に返します

友だち追加 / LINE 内 CV / LINE 経由 Web CV — どれも独立したイベントとして Meta・Google・Yahoo に Conversions API で送信。

3 つのイベント、すべて対応

友だち追加 CV

Cookie で広告クリック ID を保持。実追加数とのズレを抑えて CAPI 送信。

LINE 内 CV

LINE 内でのフォーム回答完了・リッチメニュー導線などのアクションを CV としてキャッチ。

LINE 経由 Web CV

LINE から Web に戻った後の購入・予約を、元の広告クリックに紐付けて返却。

よくあるご質問

使わない月の広告アカウント連携費はどうなりますか？

ダッシュボードから一時停止できます。停止中は課金対象外（計測は継続）。基本料金 ¥10,000 は継続します。

既存の Meta Pixel・GA4 はそのままで使えますか？

はい。独立したカスタムイベントとして送信するため、既存の計測数値を汚しません。

既存の LINE 拡張ツール（エルメ・Lステップ等）と併用できますか？

はい。Webhook を中継して既存ツールに転送する設計です。
